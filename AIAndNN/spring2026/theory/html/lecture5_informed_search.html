Lecture 5: Informed Search Strategies

AI me "smart search" ka idea: random ya blind search ki jagah, heuristic (educated guess) use karke goal tak fast pohanchna.

Heuristic h(n) Greedy Best-First Search A* Search Optimality & Completeness

Informed Search Kya Hai?

Informed search problem-specific knowledge use karta hai taake best next move choose kiya ja sake.

Heuristic = Smart Guess

Heuristic batata hai ke kisi node se goal kitna qareeb ya door ho sakta hai.

Speed vs Accuracy

Kuch algorithms fast hote hain, kuch guaranteed best answer dete hain. Trade-off samajhna zaroori hai.

Real AI Use

Navigation apps, game AI, robotics path planning me informed search bohat common hai.

1) Heuristic Function h(n)

Heuristic function node n se goal tak remaining cost ka estimate deti hai.

  • h(n) ≥ 0 hota hai.
  • Goal node par h(goal) = 0 hota hai.
  • Chhota h(n) usually matlab goal qareeb.
  • Example: Euclidean distance (straight line distance).

Simple lafzon me: heuristic aik "direction hint" hai. Ye hume batata hai kis side jana behtar lag raha hai.

2) Why Informed Search?

  • Efficiency: blind search se generally fast.
  • Direction: random explore karne ke bajaye goal ki taraf move karta hai.
  • Resource Saving: kam memory aur kam time.

Agar state space bohat bara ho, to informed search practical approach ban jata hai.

3) Greedy Best-First Search (GBFS)

GBFS sirf h(n) dekh kar node choose karta hai jiska heuristic sab se kam ho.

Selection Rule: choose node with minimum h(n)
  • Pros: fast, kam nodes explore karta hai.
  • Cons: optimal guaranteed nahi, loops ka risk agar visited handling na ho.

Greedy approach: "abhi jo best lag raha hai usi taraf chalo". Kabhi bohat achha, kabhi trap me bhi le ja sakta hai.

4) A* Search

A* Uniform Cost Search + Greedy ka combined strong version hai.

f(n) = g(n) + h(n)
  • g(n): start se n tak exact cost.
  • h(n): n se goal tak estimated cost.
  • f(n): total estimated path cost.

Admissible heuristic ho (never overestimate), to A* optimal path deta hai.

5) GBFS vs A* Quick Comparison

Property GBFS A*
Evaluation h(n) g(n) + h(n)
Speed Usually very fast Moderate (depends on heuristic)
Optimality No guarantee Yes, if heuristic admissible/consistent
Completeness May fail in tricky spaces Complete in finite spaces with proper heuristic

7) A* Ki Power: Complete + Optimal

  • Complete: finite space me solution milne ka strong chance, agar conditions satisfy hon.
  • Optimal: admissible (aur preferably consistent) heuristic ke sath shortest path deta hai.

Isi wajah se A* practical AI systems me bohat popular hai jab hume "best path" chahiye hota hai, sirf "koi path" nahi.

10) English Version (Complete Notes)

What is Informed Search? Informed search uses domain knowledge (heuristics) to guide exploration toward the goal faster than blind search.

  • Heuristic h(n): Estimated cost from node n to the goal.
  • Properties: h(n) >= 0, and h(goal) = 0.
  • GBFS: Expands node with lowest h(n). Fast, but not always optimal.
  • A*: Uses f(n)=g(n)+h(n), combining actual path cost and estimated remaining cost.
  • Optimality: A* is optimal with an admissible heuristic (never overestimates).
  • Completeness: In finite spaces with proper conditions, A* is complete.
  • Practical use: Navigation systems, robotics, game AI, and path planning.

Key intuition: GBFS follows what looks nearest now; A* balances “distance traveled so far” and “distance left,” so it usually makes better global decisions.
